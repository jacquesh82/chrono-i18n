    Namespace i18n

    Unified, language-agnostic entry point ("i18n").

    chrono ships a fully-tuned pipeline per language (chrono.fr, chrono.de, …), but the caller must know the language up front. This module runs several locales at once and merges their results, so a single call handles mixed- or unknown-language input:

    import * as chrono from "chrono-node";

    chrono.i18n.parse("réunion mardi prochain à 14h"); // → fr wins
    chrono.i18n.parse("Treffen am Dienstag");          // → de wins
    chrono.i18n.parse("riunione martedì");             // → it wins

    // Restrict the candidate set (also sets tie-break priority):
    chrono.i18n.parse(text, ref, { locales: ["fr", "en", "de"] });

    // Each result carries the locale that produced it:
    chrono.i18n.parse("demain")[0].locale; // "fr"

    Each locale's own parsers/refiners run unchanged; this layer only arbitrates between the per-locale outputs, keeping the richest, longest, non-overlapping matches.
